# OrderControl v1.1

Веб-приложение для учета товаров, остатков и движений на складе с ролевым доступом, журналом действий и документами прихода/расхода.

## Возможности

- Авторизация пользователей с ролями (`admin`, `manager`, `viewer`)
- Справочники: клиенты, категории, подкатегории, товары
- Складской контур:
  - документы прихода (`/stock/in/docs`)
  - документы расхода (`/stock/out/docs`)
  - автоматический учет движений и остатков
- Просмотр движений склада и каталога
- Логирование действий пользователей (аудит)
- Загрузка изображений товаров (JPG/PNG/WEBP)
- Простые realtime-обновления через polling (`/realtime/poll`)

## Технологии

- Python 3.10+
- Flask
- SQLite
- Jinja2 (шаблоны)
- OpenPyXL (экспорт/работа с Excel)

## Быстрый старт (Windows / PowerShell)

1. Создайте и активируйте виртуальное окружение:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

2. Установите зависимости:

```powershell
pip install -r requirements.txt
```

3. Создайте файл `.env` в корне проекта (пример ниже).

4. Запустите приложение:

```powershell
python .\flask_app.py
```

5. Откройте в браузере адрес:

- `http://127.0.0.1:80` (или порт из `APP_PORT`)

## Первая авторизация

При первом запуске автоматически создается пользователь по умолчанию:

- Логин: `root`
- Пароль: `root`

После входа сразу смените пароль в профиле и, при необходимости, создайте отдельных пользователей.

## Пример `.env`

```env
# Основное
APP_PORT=80
APP_DEBUG=0
APP_VERSION=v1.1
COOKIE_SECRET=replace-with-long-random-secret

# База данных
SQLITE_PATH=./data/ordercontrol.sqlite3

# Безопасность/сессии
SESSION_IDLE_TIMEOUT_MINUTES=60
LOGIN_MAX_ATTEMPTS=5
LOGIN_BLOCK_MINUTES=10
MAX_CONTENT_LENGTH=5242880

# Интерфейс/прочее
APP_CURRENCY=$
TZ_OFFSET_HOURS=5
BLOCK_SEARCH_INDEXING=0
STATIC_CACHE_BUSTER=v1.1
LOGS_MAX_ROWS=200000
PRODUCT_UPLOAD_DIR=./static/uploads/products
```

## Важные настройки

- `COOKIE_SECRET` - обязательно замените на длинную случайную строку
- `APP_DEBUG=0` - рекомендуется для production
- `SQLITE_PATH` - путь к файлу SQLite (папка создается автоматически)
- `PRODUCT_UPLOAD_DIR` - папка загрузки изображений товаров
- `MAX_CONTENT_LENGTH` - максимальный размер запроса/файла в байтах

## База данных

Приложение само:

- создает SQLite-файл и нужные папки;
- применяет схему таблиц при старте;
- выполняет мягкие миграции по колонкам;
- создает дефолтного администратора, если таблица `users` пустая.

## Деплой (кратко)

Для хостинга с Passenger используйте `passenger_wsgi.py` как WSGI entrypoint.

Минимальные рекомендации для production:

- включить HTTPS;
- задать безопасный `COOKIE_SECRET`;
- использовать `APP_DEBUG=0`;
- настроить резервное копирование SQLite файла.

## Полезные разделы в интерфейсе

- `/admin` - дашборд
- `/catalog` - каталог и быстрые действия
- `/stock/in/docs` - документы прихода
- `/stock/out/docs` - документы расхода
- `/stock/moves` - движения по складу
- `/logs` - журнал событий
- `/admins` - пользователи и роли

## Функции по страницам

- `/login` - вход в систему, проверка логина/пароля, контроль блокировок по попыткам
- `/admin` - главная панель: быстрый обзор системы и переходы в рабочие разделы
- `/clients` - список клиентов, поиск/фильтрация, добавление нового клиента
- `/clients/<id>` - редактирование карточки клиента, включение/отключение активности
- `/categories` - список категорий, добавление новой категории
- `/categories/<id>` - редактирование названия категории
- `/subcategories` - список подкатегорий, добавление подкатегории с выбором категории
- `/subcategories/<id>` - редактирование подкатегории и ее привязки к категории
- `/products` - список товаров, добавление товара (SKU, единица, мин. остаток, активность)
- `/products/<id>` - редактирование товара, загрузка/обновление изображения, изменение параметров
- `/catalog` - работа с номенклатурой в каталоге, быстрые переходы к созданию категорий/подкатегорий
- `/stock/in/docs` - список документов прихода (черновики и проведенные)
- `/stock/in` - создание/редактирование документа прихода, добавление позиций и количества, проведение документа
- `/stock/out/docs` - список документов расхода (черновики и проведенные)
- `/stock/out` - создание/редактирование документа расхода, выбор клиента, добавление позиций, контроль остатка, проведение
- `/stock/moves` - журнал движений товаров (приход/расход), фильтры и контроль истории операций
- `/logs` - аудит-действия пользователей (кто, что, когда изменил)
- `/admins` - список пользователей системы, создание нового пользователя, назначение роли и активности
- `/admins/<id>` - редактирование пользователя, прав и статуса
- `/profile` - профиль текущего пользователя, смена пароля
- `/logout` - завершение сессии
